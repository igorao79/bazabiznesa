## Запрос #1 — Инициализация проекта

**Дата:** 2026-03-10, 21:00
**Цель:** Создание полной архитектуры проекта по ТЗ

**Промпт:**

```
Реализуй веб-сервис "Заявки в ремонтную службу" на основе следующего ТЗ.

Стек: Node.js + Express + TypeScript (backend), React + TypeScript + Vite + Tailwind CSS (frontend),
Prisma ORM, PostgreSQL, Docker Compose.

Функциональные требования:
- Две роли: диспетчер и мастер. Авторизация через JWT (логин/пароль).
- Сущность Request: clientName, phone, address, problemText, status (new | assigned | in_progress | done | canceled), assignedTo, createdAt, updatedAt.
- Три страницы: создание заявки, панель диспетчера (список, фильтры, назначение мастера, отмена), панель мастера (список назначенных, "Взять в работу", "Завершить").
- Race condition protection: действие "Взять в работу" должно быть атомарным — при параллельных запросах один успешен, второй получает 409 Conflict. Реализовать через optimistic concurrency control (поле version).

Технические требования:
- Docker Compose с тремя сервисами (db, backend, frontend+nginx).
- Prisma миграции и сиды: 1 диспетчер, 2 мастера, 5 тестовых заявок.
- Минимум 2 автотеста (vitest + supertest).
- Скрипт race_test.sh для проверки параллельного "взятия в работу".
- Файлы README.md, DECISIONS.md, PROMPTS.md.

Дополнительные фичи (сверх ТЗ):
- Audit log: история изменений статуса каждой заявки с записью кто/когда/что изменил.
- Приоритеты заявок (low, normal, high, urgent) с визуальной индикацией.
- Комментарии к заявкам.
- Статистика на панели диспетчера (количество заявок по статусам).
- Поиск заявок по клиенту, адресу, телефону.

Используй Context7 MCP для получения актуальной документации по Prisma, Express, React.
```

**Результат:** Сгенерирована полная структура проекта — 53 файла, включая бэкенд с 3-слойной архитектурой (routes → services → prisma), фронтенд с 7 страницами/компонентами, Docker-инфраструктуру, миграции, сиды, тесты.

## Запрос #2 — Запуск локального окружения

**Дата:** 2026-03-10, 21:15
**Цель:** Запуск серверов для локальной разработки

**Промпт:**

```
Запусти сервер. Docker и PostgreSQL не установлены локально —
адаптируй проект для работы с SQLite без Docker.
```

**Результат:**
- Prisma provider переключён с PostgreSQL на SQLite.
- Enum-типы заменены на строковые поля (SQLite не поддерживает enum).
- `createMany` заменён на отдельные `create` (SQLite не поддерживает batch insert).
- Удалён `mode: "insensitive"` из поисковых запросов (нет поддержки в SQLite).
- Серверы запущены: backend на порту 3001, frontend на порту 5173.
